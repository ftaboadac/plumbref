# Alternatives And Substitutes

Plumbref should be judged against existing workflows, not against a careless
agent.

Important substitutes:

- `rg` plus manual source reading
- asking the coding agent to cite files and line ranges
- asking the coding agent to perform contradiction searches
- custom `AGENTS.md` instructions or project rules
- tests, CI, type checking, and linters
- Semgrep or other static analysis
- code review checklists
- generated docs or architecture notes
- careful use of Codex, Claude Code, Cursor, or similar tools without Plumbref

The strongest differentiation claim to test:

> Plumbref makes the verification workflow explicit, repeatable, and inspectable
> instead of relying on the agent to voluntarily behave carefully.
